# 小红书关注列表迁移工具 - 使用说明

## 快速开始

### 1. 安装依赖

```bash
cd xiaohongshu-follow-migration
pip install -r requirements.txt
playwright install chromium
```

### 2. 导出关注列表

```bash
# 导出旧账号的关注列表
python scripts/export_following.py --account old_account

# 按提示操作：
# 1. 浏览器会自动打开小红书登录页面
# 2. 手动登录旧账号
# 3. 按回车键继续
# 4. 等待脚本自动滚动加载所有关注用户
```

### 3. 批量关注新账号

```bash
# 批量关注到新账号（分批进行，避免风控）
python scripts/batch_follow.py --account new_account --file data/following_old_account_*.csv --batch-size 50 --delay 10

# 参数说明：
# --batch-size 50  每批关注50人
# --delay 10       批次间延迟10秒
```

**注意：老账号不会取消关注，仅在新账号上关注。**

## 参数说明

### export_following.py

| 参数 | 说明 | 示例 |
|------|------|------|
| `--account` / `-a` | 账号名称（用于标识） | `--account old_account` |
| `--output` / `-o` | 输出目录 | `--output data` |

### batch_follow.py

| 参数 | 说明 | 示例 |
|------|------|------|
| `--account` / `-a` | 账号名称 | `--account new_account` |
| `--file` / `-f` | CSV文件路径 | `--file data/following_old.csv` |
| `--batch-size` / `-b` | 每批关注数量 | `--batch-size 50` |
| `--delay` / `-d` | 批次间延迟（秒） | `--delay 10` |
| `--output` / `-o` | 输出目录 | `--output data` |

**注意：`unfollow.py` 脚本仍保留在项目中，但默认不使用。如需取消老账号关注，可手动运行。**

## 安全建议

### 1. 分批操作

- **每天关注不超过100人**
- **每批50人左右**
- **批次间延迟10秒以上**

### 2. 风控处理

- 出现验证码时暂停操作
- 手动完成验证码后继续
- 避免频繁切换账号

### 3. 数据安全

- CSV文件包含用户信息，请妥善保管
- 不要将CSV文件上传到公开仓库
- 操作完成后及时删除CSV文件

## 常见问题

### Q1: 为什么不能一次性关注900+人？

**A:** 小红书有严格的风控机制，一次性关注大量用户会触发验证码甚至封号。建议每天关注100人以内。

### Q2: 出现验证码怎么办？

**A:** 脚本会自动暂停并提示用户手动完成验证。完成后按回车键继续即可。

### Q3: 如何确保数据安全？

**A:** CSV文件只保存在本地，操作完成后建议手动删除。不要将CSV文件上传到公开仓库。

### Q4: 可以在不同设备上操作吗？

**A:** 可以。导出CSV后，将CSV文件复制到另一台设备，然后在新账号上执行批量关注。

### Q5: 如何验证关注是否成功？

**A:** 脚本会生成详细的操作日志和统计文件。可以查看日志文件确认操作结果。

## 注意事项

1. **首次运行需要安装Playwright：**
   ```bash
   playwright install chromium
   ```

2. **操作前请备份数据：**
